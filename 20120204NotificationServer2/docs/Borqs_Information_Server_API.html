播思通知服务API v1.3.0

  1. 术语与约定

  • {host}：指通知服务具体部署的主机IP或者域名。

  • {port}：指通知服务的访问端口号。

  • {ticket}：身份认证信息。

  • {mid}：消息ID


  • (暂不处理)：当前不需要处理的内容，但以后可能需要处理的内容。

  • (暂时使用)：是指为了当前开发需要保留的内容，应用开发完成后具有此标记的内容可能会被移除。

  • 请求参数列表中，加粗行为当前需要实现的参数。

  1. 接口及说明

  1. informations/send

功能：用户发送一条通知

地址：http://{host}:{port}/bmb/service/informations/send.json

响应数据格式：JSON

HTTP请求方式：POST

权限控制：是

请求参数：

Name

Type

Required

Range

Remark

ticket

String

True


身份验证信息


请求示例

请求URL

http://{host}:{port}/bmb/service/informations/send.json?ticket={ticket}


请求数据

{"appId":"testapp","receiverId":"123","title":"this is a test title", "type":"type1"}


返回结果

{"result":{"status":"success"}}

或者

{"result":{"status":"failed"}}
