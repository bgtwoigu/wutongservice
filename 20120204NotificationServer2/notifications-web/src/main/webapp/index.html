播思通知服务API v0.4

一、术语与约定

{host}：指通知服务具体部署的主机IP或 者域名。
{port}：指通知服务的访问端口号。
{ticket}：身份认证信息。

(暂不处理)：当前不需要处理的内容，但以后可能需要处理的内容。

(暂时使用)：是指为了当前开发需要保留的内容，应用开发完成后具有此标记的内容可能会被移除。

请求参数列表中，加粗行为当前需要实现的参数。

三、接口及说明

1. informations/send

功能： 用户发送一条通知

地址：http://{host}:{port}/bmb/service/informations/send.json

响应数据格式：JSON

HTTP请求方式 ：POST

权限控制： 是

请求参数：

Name

Type

Required

Range

Remark

ticket

String

True

身份验证信息

请求示例

请求URL

http://{host}:{port}/bmb/service/informations/send.json?ticket={ticket}

请求数据

{"appId":"testapp","receiverId":"123","title":"this is a test title", "type":"type1","action":"action.test", "data":"this is test data."}

返回结果

{"result":{"status":"success"}}

或者

{"result":{"status":"failed"}}

功能：获 取用户通知列表

地址：http://{host}:{port}/bmb/service/informations/list.json

响应数据格式：JSON

HTTP请求方式 ：GET

权限控制： 是

请求参数：

Name

Type

Required

Range

Remark

String

True

身份验证信息

status

int

False

0或者1

0－表示未处理（默认）；1－ 表示已处理

Int

False

从第几条通知开始取，默认为0

size

Int

False

取多少条通知，默认为20, -1时取所有符合条件的通知
